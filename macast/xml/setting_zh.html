常见问题 (FAQ)

本文档可能不是最新版本。请访问 Macast Wiki 获取最新文档。

Macast 仍处于开发早期阶段，因此可能在您的设备上不可用。如果遇到这种情况，请尝试 测试版 或提交 issue 来帮助我们做得更好。

我不知道该如何使用

只需打开此应用，状态栏 / 任务栏 / 桌面面板上就会出现一个小图标，然后您就可以从本地 DLNA 客户端（例如手机上的视频软件）将媒体文件投屏到电脑上。

如何使用第三方播放器插件

  1. 从 xfangfang/Macast-plugins 下载您需要的插件
  2. 打开 Macast，点击设置菜单中的 打开配置目录
  3. 将 [某个插件].py 放入 renderer 目录下
  4. 重启 Macast 并选择您需要的插件

如果找不到您需要的插件，可以查看 这里 学习如何编写自定义播放器插件，非常欢迎您提交 Pull Request。

无法从您的 DLNA 客户端中找到 Macast

  • 检查您的防火墙是否拦截了 Macast。
    一个简单的检查防火墙方法是从其他设备访问 Macast 打开的端口（默认是 1068，您可以在设置菜单中找到它）
  • 检查您的路由器。
    您应当开启 UPnP 并关闭 AP 隔离。
  • 软件问题。
    如果您使用的是 iOS 设备，请确保已打开该应用 查找并连接本地网络上的设备 的权限。
    另外，在某些应用中，您需要等待 30 秒以上才能找到 Macast，请耐心等待。
  • UDP 组播问题。
    据说 Windows 和某些路由器会存在适配问题，导致 Windows 无法加入组播。在这种情况下，您需要修改 Windows 注册表以强制将 IGMP 版本更改为 v2。
  • 多网卡或复杂的网络状况。
    确保 Macast 在单网卡以及与您的 DLNA 客户端在同一局域网下可以使用。对于更复杂情况的解决方案仍在开发中。

启动时崩溃

Macast 默认打开 1068 端口。当您的电脑上存在端口冲突时，应用会崩溃。请手动设置应用端口，然后 Macast 将会从配置文件中加载您指定的端口。

无法关闭或无响应

使用第三方播放器插件时，插件通常需要打开更多的线程或进程，导致应用退出时出现无响应的问题，这通常是由插件本身引起的。

并在配置文件中填入一些 mpv 命令行选项，例如：

fs-screen==1

此选项会让 mpv 在屏幕 1 上全屏打开新视频。了解更多关于 fs-screen 的选项： https://mpv.io/manual/stable/#options-fs-screen
